# MDX in Astro

MDX is Markdown with superpowers — you can import and use **Astro components** (or React, Vue, Svelte) directly inside your prose, no extra configuration needed per-file.

## Why MDX?

Plain Markdown is great for text, but sometimes you need richer UI inline with your writing:

- Interactive code playgrounds
- Custom callout boxes
- Data-driven charts
- Anything a component can do

<Callout type="info">
  MDX files use the `.mdx` extension and are picked up automatically by the content collection glob `**/*.{md,mdx}`.
</Callout>

## Importing Components

At the top of any `.mdx` file, add a standard JS import **below the frontmatter fence**:

```mdx
import Callout from '../../components/Callout.astro';
```

Then drop the component anywhere in the body like regular JSX:

```mdx
<Callout type="warning">
  Watch out — this is a destructive operation!
</Callout>
```

## How the Component Works

The `Callout` component (`src/components/Callout.astro`) accepts two props:

| Prop    | Type                                      | Default  |
| ------- | ----------------------------------------- | -------- |
| `type`  | `'info' \| 'warning' \| 'error' \| 'success'` | `'info'` |
| `title` | `string`                                  | auto     |

It renders a DaisyUI `alert` with `role="alert"` and `not-prose` so Tailwind Typography doesn't fight the alert styles.

## Wrapping Up

MDX + Astro is a clean combo: your content stays readable as text, but you unlock component-level expressiveness whenever you need it.
